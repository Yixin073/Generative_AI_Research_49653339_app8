# AI_Research

This repository contains research materials for the project "Using Generative AI Tools - Boon or Bane". The files are organised into folders based on the type of material and the stage of the research project.

## Repository Structure

- `data`: stores the main project data.
  - `raw_data`: stores original survey data, such as `raw_data_v1.csv` and `extra_raw_data_v1.csv`.
  - `cleaned_data`: stores cleaned data and data-cleaning files, such as `cleaned_data_v1.csv` and `python_for_cleaning_v1.py`.

- `drafts`: stores draft documents, such as `proposal_draft_v1.docx`.

- `paper_submit`: stores submitted versions of the proposal and paper, such as `proposal_submit_v1.docx` and `paper_submit_v1.pdf`.

- `qualitative analysis`: stores qualitative materials, such as interview transcripts and survey question documents.

- `quantitative analysis`: stores quantitative analysis outputs, such as charts, tables, and workflow diagrams.

- `reference`: stores academic sources and background references used for the project.

## Naming Convention

Files in this repository use simple and consistent names with lowercase words, underscores, and version numbers. The main format is `file_content_v1`, such as `raw_data_v1.csv`, `cleaned_data_v1.csv`, `survey_questions_v1.docx`, and `paper_submit_v1.pdf`. This makes it easier for team members to understand the file content and track updated versions.

## Access Control

Some files in this project may contain sensitive research information, such as interview transcripts, survey responses, or raw data. These files should have restricted access and should not be shared publicly unless they are anonymised and approved for sharing.

## Collaboration Guidelines

Team members should use clear commit messages when updating files, so the project history is easy to understand. If a major change is needed, it should be made in a separate branch and reviewed before being merged into the main branch. This helps reduce confusion when multiple people are working on the same project.
